1. Explain the Angular workflow.

The Angular workflow describes the process of creating, building, and deploying Angular applications.

It typically involves steps such as setting up the development environment, creating components and services, writing code, testing, and optimizing for production.

2. List down all Life Cycle Hooks in Angular

Angular provides 8 life cycle hooks. These are:

  • ngOnChanges
  • ngOnInit
  • ngDoCheck
  • ngAfterContentInit
  • ngAfterContentChecked
  • ngAfterViewInit
  • ngAfterViewChecked
  • ngOnDestroy

3. Explain component communication in Angular.

Component communication in Angular involves sharing data between Angular components.

There are several methods for component communication, including Input and Output properties, ViewChild and ContentChild decorators, and shared services.

4. Explain the difference between ngIf and ngShow in Angular.

ngIf and ngShow are both Angular directives used for conditionally displaying elements in the DOM.

  • ngIf: Removes or recreates the element in the DOM based on a condition. When the condition is false, the element is removed from the DOM. When the condition is true, the element is added to the DOM.
  • ngShow: Hides or shows the element based on a condition using CSS display property. When the condition is false, the element is hidden by setting its display property to none. When the condition is true, the element is shown by setting its display property to its original value.

5. Explain Angular pipes.

Angular pipes are used for data transformation within templates.

They allow developers to transform data before displaying it in the view. Angular provides several built-in pipes for common tasks such as formatting dates, numbers, and currency.

6. Explain Injectable and Singleton Services in Angular.

Injectable services in Angular are classes that are decorated with the Injectable() decorator.

Singleton services are services that are instantiated only once per application and are shared across the entire application.

7. Explain Dependency Injection in Angular.

Dependency Injection (DI) is a design pattern in which a class receives its dependencies from external sources rather than creating them itself.

In Angular, DI is used to provide instances of services to components, directives, pipes, and other services.

8. Explain host listener and host binding in Angular.

Host listener and host binding are Angular decorators used to listen to events and bind properties on the host element of a directive.

Host listener is used to listen to events such as mouse clicks, keyboard events, etc., while host binding is used to bind properties of the host element.

9. Explain one-way and two-way data binding in Angular.

One-way data binding refers to the flow of data from the component class to the template (view).

Two-way data binding refers to the synchronization of data between the component class and the template. Changes in the template update the component class and vice versa.

10. Explain Route/Auth Guards in Angular.

Route guards in Angular are used to control access to routes in an Angular application.

Auth guards specifically ensure that only authenticated users are allowed to access certain routes. They can be used to redirect unauthorized users to a login page or display a forbidden message.

11. Explain Lazy Loading in Angular.

Lazy loading is a technique in Angular where modules are loaded asynchronously, on-demand, rather than upfront when the application starts.

It helps improve the initial loading time of the application by only loading the necessary modules when they are required.

12. Explain Template/Reactive Forms and Validations in Angular.

Template-driven forms and reactive forms are two approaches for handling forms in Angular.

Template-driven forms rely on directives in the template, while reactive forms use reactive programming with FormControl, FormGroup, and FormBuilder classes.

13. Explain Observable and Promises in Angular.

Observable and Promises are both used for handling asynchronous operations in Angular.

Observables are streams of data that can be subscribed to, while Promises represent a single future value or error that will be resolved asynchronously.

14. Explain RXJS Operators in Angular.

RXJS Operators are functions used to manipulate the data emitted by Observables.

They allow developers to perform operations such as filtering, mapping, merging, and combining Observables to create new streams of data.

15. Explain Subject and Behavior Subject in Angular.

Subject and Behavior Subject are types of Observables used for multicasting.

Subject is a special type of Observable that allows values to be multicast to multiple Observers, while Behavior Subject is a type of Subject that stores the latest value and emits it to new subscribers.

16. Explain HTTP (Req/Res) Interceptor in Angular.

HTTP Interceptors in Angular are middleware that intercept HTTP requests and responses.

They are used to modify or transform requests or responses globally in an Angular application.

17. Explain Change Detection and OnPush in Angular.

Change Detection in Angular is the process of detecting changes to the application state and updating the view accordingly.

OnPush is a change detection strategy in Angular that triggers change detection only when input properties of a component change or when an event is emitted from the component.

18. Explain dist and bundle in Angular.

Dist and bundle refer to the distribution and bundling of Angular applications.

When an Angular application is built for production, it is compiled, minified, and bundled into one or more JavaScript files that are ready for deployment.
